## SEO Positioning & Goals

### Search Positioning

TaskClaw occupies a unique niche at the intersection of three search verticals:

```
[Open-Source Project Management] + [AI Task Automation] + [Self-Hosted Tools]
```

Our SEO strategy targets users searching for:
1. **Alternatives** to existing task managers (Notion, ClickUp, Asana)
2. **AI-powered** productivity and automation tools
3. **Self-hosted** and open-source project management
4. **Task orchestration** and multi-tool sync solutions

### SEO Goals (12-month)

| Goal                                          | Target                  |
|-----------------------------------------------|-------------------------|
| Organic monthly visitors                      | 10,000+                 |
| Domain authority (Ahrefs DR)                  | 30+                     |
| Ranking keywords (top 20)                     | 200+                    |
| Featured snippets captured                    | 10+                     |
| GitHub traffic from organic search            | 20% of total repo visits |

## On-Page SEO Standards

### Title Tag Formula

```
[Primary Keyword] - [Benefit/Descriptor] | TaskClaw
```

**Examples:**
- `Open Source AI Task Management - Sync All Your Tools | TaskClaw`
- `Self-Hosted Kanban Board with AI Execution | TaskClaw`
- `How to Sync Notion and ClickUp Tasks in One Board | TaskClaw`

**Rules:**
- Maximum 60 characters (including `| TaskClaw`)
- Primary keyword within the first 5 words
- Include the brand name at the end
- Every page must have a unique title

### Meta Description Formula

```
[Problem or hook]. [Solution with TaskClaw]. [CTA or differentiator]. [Max 155 chars]
```

**Examples:**
- `Tired of switching between Notion, ClickUp, and Trello? TaskClaw syncs all your tasks into one Kanban board. Open-source & self-hosted.`
- `TaskClaw is an open-source task hub that lets AI start your work. Sync tools, assign AI skills, and ship faster. Self-hosted, privacy-first.`

**Rules:**
- 120-155 characters
- Include primary keyword naturally
- End with a differentiator or CTA
- No keyword stuffing — write for humans

### Heading Structure (H1-H4)

Every page must follow this hierarchy:

```
H1: One per page. Contains primary keyword. Matches page intent.
  H2: Section headings. Contains secondary keywords where natural.
    H3: Subsections. Descriptive, scannable.
      H4: Rare. Only for deep technical content.
```

**Rules:**
- Only one H1 per page
- H2s should be scannable — someone skimming headings should understand the page
- Include keywords naturally; never force them
- Use question-format H2s for FAQ/educational content (triggers featured snippets)

### URL Structure

```
taskclaw.dev/[category]/[topic-keyword]
```

**Examples:**
- `/docs/getting-started`
- `/blog/sync-notion-clickup-one-board`
- `/compare/taskclaw-vs-notion`
- `/guides/setup-openclaw-vps`

**Rules:**
- Lowercase, hyphen-separated
- 3-5 words maximum
- Include primary keyword
- No dates in URLs (content should feel evergreen)
- No trailing slashes

### Image SEO

| Attribute     | Standard                                           |
|---------------|-----------------------------------------------------|
| File name     | Descriptive, hyphenated: `taskclaw-kanban-dark.png` |
| Alt text      | Describe the image for accessibility + include keyword naturally |
| Format        | WebP for web, PNG for screenshots, SVG for icons    |
| Size          | < 200KB for web images, use lazy loading            |
| Dimensions    | Specify width/height to prevent layout shift (CLS)  |

### Content Types & Funnel Mapping

| Funnel Stage     | Content Type          | Goal                          | Keyword Focus             |
|------------------|-----------------------|-------------------------------|---------------------------|
| **Awareness**    | Blog posts            | Attract organic traffic       | Problem-aware keywords    |

| **Interest**     | Feature guides        | Showcase capabilities         | Feature + benefit keywords |

| **Decision**     | Self-hosting guide    | Reduce setup friction         | Setup + deployment keywords |
| **Retention**    | Changelog / Updates   | Keep users engaged            | Brand keywords            |
| **Advocacy**     | Contributor guides    | Grow the community            | OSS contribution keywords |

### Blog Content Frameworks

#### Framework 1: Problem-Agitate-Solve (PAS)

Best for: Awareness content, pain-point articles

```
Introduction: Identify a specific problem the reader has
Agitate: Describe why current solutions fail (with examples)
Solve: Introduce TaskClaw's approach as the solution
How-to: Practical steps to implement
CTA: Try TaskClaw / Star on GitHub
```

**Example article:** "Why Your Task Manager Is Making You Less Productive (And What to Do About It)"

#### Framework 2: Tutorial / Step-by-Step

Best for: Consideration content, documentation-adjacent posts

```
What you'll build/learn (outcome-first)
Prerequisites (keep this short)
Step 1-N (with code blocks and screenshots)
What's next (link to related content)
```

**Example article:** "Sync Notion and ClickUp into One Kanban Board in 10 Minutes"

#### Framework 3: Comparison / "Versus"

Best for: Decision-stage content, capturing competitor traffic

```
Introduction: What both tools do
Feature comparison table (honest — include where TaskClaw loses)
Use case scenarios (when to use each)
Conclusion: Who TaskClaw is best for
```

**Example article:** "TaskClaw vs. Notion: When You Need More Than a Workspace"

**Rules for comparison content:**
- Be honest about competitor strengths
- Never trash-talk — compare features factually
- Focus on where TaskClaw's approach (self-hosted, AI execution) creates unique value
- Update regularly as competitors evolve

#### Framework 4: Listicle / Roundup

Best for: Awareness content, attracting broad keyword traffic

```
Introduction: Why this list matters
Items 1-N (with brief descriptions and TaskClaw mention where relevant)
Summary / Key takeaways
```

**Example article:** "7 Best Open Source Task Management Tools for Developers in 2026"

## Technical SEO Checklist

### Core Web Vitals Targets

| Metric                          | Target              | Tool                           |
|---------------------------------|---------------------|--------------------------------|
| Largest Contentful Paint (LCP)  | < 2.5s              | Lighthouse, PageSpeed Insights |
| First Input Delay (FID)         | < 100ms             | Chrome UX Report               |
| Cumulative Layout Shift (CLS)   | < 0.1               | Lighthouse                     |
| Time to First Byte (TTFB)      | < 600ms             | WebPageTest                    |

### Technical Requirements

- [ ] **SSL/HTTPS** on all pages
- [ ] **Mobile responsive** — all pages must work on 320px+ screens
- [ ] **Sitemap.xml** — auto-generated, submitted to Google Search Console
- [ ] **robots.txt** — allow all crawling except admin/dashboard routes
- [ ] **Canonical URLs** — set on every page to prevent duplicate content
- [ ] **Structured data (JSON-LD)** — SoftwareApplication schema on homepage, Article schema on blog posts
- [ ] **Open Graph tags** — title, description, image for every page
- [ ] **Twitter Card meta** — summary_large_image for all pages
- [ ] **Hreflang tags** — when i18n content is available
- [ ] **404 page** — custom, with navigation back to useful pages
- [ ] **Redirect handling** — 301 redirects for any moved pages
- [ ] **Page speed** — images optimized, lazy loaded, scripts deferred

### Tier 2: Content-Driven Links

| Strategy                           | How                                                    |
|------------------------------------|--------------------------------------------------------|
| Guest posts                        | Write for dev blogs about AI task automation            |
| Open-source roundups               | Pitch inclusion in "best OSS tools" articles           |
| Tutorial cross-posting             | Publish guides on Dev.to, Hashnode, Medium (canonical) |
| GitHub integrations                | Build plugins/connectors that reference TaskClaw       |
| Conference talks                   | Submit to DevOps/productivity conferences              |

### Tier 3: Community & PR

| Strategy                           | How                                                    |
|------------------------------------|--------------------------------------------------------|
| Reddit (organic)                   | Genuine participation in r/selfhosted, r/opensource    |
| Discord communities                | Engage in productivity/dev Discord servers             |

### Link Building Rules

- **Never** buy links or participate in link schemes
- **Always** add value — don't drop links without context
- **Prioritize** relevant, high-quality sites over volume
- **Track** all backlinks in a spreadsheet or Ahrefs

## Content Templates

### Blog Post SEO Template

```markdown
---
title: "[Primary Keyword] - [Benefit] | TaskClaw"
description: "[120-155 char meta description with primary keyword]"
date: YYYY-MM-DD
author: "[Author Name]"
tags: ["tag1", "tag2", "tag3"]
image: "/blog/images/[descriptive-filename].webp"
canonical: "https://taskclaw.dev/blog/[url-slug]"
---

# [H1: Primary keyword + value proposition]

[Opening paragraph: Hook the reader with a relatable problem or surprising fact. 2-3 sentences max.]

## [H2: Problem/Context Section]

[Expand on the problem. Use data, examples, or anecdotes. 150-300 words.]

## [H2: Solution/How-to Section]

[Present the solution. If tutorial, start step-by-step here. 300-500 words.]

### [H3: Sub-steps or details]

[Code blocks, screenshots, or detailed instructions.]

## [H2: Results/Benefits]

[What the reader gains from this approach. 100-200 words.]

## [H2: Conclusion / Next Steps]

[Summarize key takeaways. 2-3 bullet points.]

[CTA: Link to GitHub, docs, or related content.]
```
