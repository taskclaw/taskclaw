## Content Pillars

Every piece of content should fall into one of these five pillars. Aim for a balanced mix across the week.

### 1. Product & Features (30%)

Showcase what TaskClaw does. Demo videos, feature spotlights, UI screenshots.

**Examples:**
- "Did you know you can assign different AI skills per task category? Marketing tasks auto-load your copywriter. Dev tasks load your code reviewer. Zero prompting."
- Short screen recording of dragging a task to the AI column
- Before/after: scattered tools vs. one unified board

### 2. Open Source & Community (25%)

Celebrate contributors, share project milestones, encourage participation.

**Examples:**
- "Thanks to @contributor for adding Linear integration! Your first PR merged."
- "We just hit 1,000 stars on GitHub. Every star is someone who's tired of glorified to-do lists."
- "Good first issues are up. If you've ever wanted to contribute to an AI-powered task tool, now's your chance."

### 3. Developer Education (20%)

Tutorials, how-tos, tips for setting up and extending TaskClaw.

**Examples:**
- "How to set up TaskClaw with your own OpenClaw instance in 5 minutes (thread)"
- "Creating custom AI skills for your Kanban categories — a quick guide"
- "Self-hosting TaskClaw on a $5 VPS: a step-by-step walkthrough"

### 4. Productivity & Pain Points (15%)

Relatable content about task management frustrations. Problem-first messaging.

**Examples:**
- "You have 4 task apps open. Your AI assistant needs a prompt every time. Your backlog is growing. Sound familiar?"
- "The average knowledge worker switches between 9 apps per day. TaskClaw says: what if it was one?"
- "Starting is always the hardest part. What if your tasks started themselves?"

### 5. Behind the Scenes (10%)

Team/contributor updates, development progress, architectural decisions.

**Examples:**
- "Here's why we chose NestJS for the backend (and why it matters for plugin developers)"
- "Working on two-way sync with Linear. Here's a sneak peek at the architecture."
- "This week's focus: improving the Pomodoro timer and adding keyboard shortcuts"

## Post Templates & Frameworks

### Template 1: Problem-Solution Hook

```
[Relatable pain point in 1-2 sentences]

[Transition: "That's why we built..." / "What if..." / "Enter TaskClaw."]

[Solution in 1-2 sentences]

[CTA: link, try it, star on GitHub]
```

**Example:**
> You spend more time managing tasks than doing them. Context-switching, copy-pasting, re-prompting your AI every single time.
>
> TaskClaw syncs everything into one board. Drag to the AI column. Done.
>
> Try it: [link]

### Template 2: Feature Spotlight

```
[Feature name] is live.

[What it does in 1 sentence]
[Why it matters in 1 sentence]

[Screenshot or GIF]

[CTA]
```

**Example:**
> Knowledge Databases are live.
>
> Attach docs, SOPs, and brand guidelines to task categories. Your AI always has the right context — no manual prompting.
>
> [screenshot]
>
> Set it up: [docs link]

### Template 3: Contributor Shoutout

```
Shoutout to @[contributor] for [what they did]!

[Brief description of the contribution and why it matters]

[Link to PR or feature]

Open source is better when people show up. Want to contribute? [link to good-first-issues]
```

### Template 4: Thread / Tutorial

```
Tweet 1: [Hook — what you'll learn + why it matters]

Tweet 2-N: [Step-by-step with code snippets or screenshots]

Final tweet: [Summary + CTA to try it or read the full guide]
```

### Template 5: Milestone / Celebration

```
[Number/achievement].

[What it means to the team / community]

[Thank the community]

[What's next]
```

**Example:**
> 500 stars on GitHub.
>
> That's 500 people who believe task managers should do more than hold lists.
>
> Thank you to every contributor, stargazer, and early adopter. This is just the beginning.
>
> Next up: Asana integration and a plugin system.

### Template 6: Hot Take / Opinion

```
[Bold, slightly provocative statement]

[1-2 sentences of reasoning]

[Tie it back to TaskClaw's philosophy]

[Optional: Ask for community opinion]
```

**Example:**
> Your task manager is a glorified to-do list.
>
> It holds things you already know you need to do. It doesn't help you start. It doesn't reduce context-switching. It just... sits there.
>
> We think task tools should actually do tasks. Agree? Disagree?

## Visual Standards for Social

### Image Dimensions

| Platform        | Post Image       | Story/Reel       | Profile Banner    |
|-----------------|------------------|-------------------|-------------------|
| **Twitter/X**   | 1200 x 675px     | N/A               | 1500 x 500px      |
| **LinkedIn**    | 1200 x 627px     | N/A               | 1584 x 396px      |
| **Discord**     | Any (max 8MB)    | N/A               | 960 x 540px       |
| **YouTube**     | 1280 x 720px     | 1080 x 1920px     | 2560 x 1440px     |
| **Reddit**      | 1200 x 628px     | N/A               | 4000 x 128px      |

### Social Post Design Rules

1. **Background:** Default to dark theme (`#0A0A0B`) for all social graphics — it matches the brand and stands out in feeds
2. **Accent glow:** Add subtle brand gradient glow behind key elements
3. **Text on images:** Use Inter Bold. Maximum 8-10 words for headline text on images
4. **Screenshots:** Always use the dark mode of the product. Add a subtle rounded border (`#FFFFFF10`) and drop shadow
5. **Logo placement:** Bottom-right corner of all branded graphics, using the dark variant
6. **Mascot usage:** Clawster can appear in social graphics but should not overlap with text
7. **Code snippets:** Use dark syntax-highlighted code blocks with JetBrains Mono font
8. **No stock photos:** Use product screenshots, illustrations, or text-based graphics only

### Color Usage in Social Graphics

| Element                  | Color                                       |
|--------------------------|---------------------------------------------|
| Background               | `#0A0A0B` (dark) or `#FFFFFF` (light)       |
| Headline text             | `#FFFFFF` (on dark) or `#0F172A` (on light) |
| Accent / highlight        | Brand Gradient (`#A855F7` to `#3B82F6`)     |
| Secondary text            | `#94A3B8`                                   |
| Feature card backgrounds  | `#FFFFFF0D` (white at 5% opacity)           |
| Border / separator        | `#FFFFFF1A` (white at 10% opacity)          |
| Claw Red accent           | `#E63B3B` (sparingly — badges, icons)       |

### GIF / Video Standards

- **Format:** MP4 for videos, GIF or WebM for short loops
- **Duration:** Under 60 seconds for social clips, under 15 seconds for GIFs
- **Framerate:** 30fps minimum
- **Always show:** The Kanban board in action — drag-and-drop, AI execution
- **Caption:** Always add captions/subtitles to videos (many watch on mute)

## Hashtag Strategy

### Primary Hashtags (Always use 2-3)

| Hashtag               | Usage                           |
|-----------------------|---------------------------------|
| `#TaskClaw`           | Every post (brand hashtag)      |
| `#OpenSource`         | All OSS-related content         |
| `#SelfHosted`         | Infrastructure/privacy content  |

### Secondary Hashtags (Rotate based on content)

| Category            | Hashtags                                                         |
|---------------------|------------------------------------------------------------------|
| **AI / Automation** | `#AIAutomation` `#AITools` `#DevTools` `#AIProductivity`         |
| **Productivity**    | `#Productivity` `#TaskManagement` `#GetThingsDone` `#Kanban`     |
| **Dev Community**   | `#BuildInPublic` `#IndieHacker` `#DevCommunity` `#100DaysOfCode`|
| **Privacy**         | `#PrivacyFirst` `#DataOwnership` `#SelfHosted`                  |
| **Tech Stack**      | `#NestJS` `#NextJS` `#TypeScript` `#React` `#TailwindCSS`       |

### Hashtag Rules

1. **Twitter/X:** 2-4 hashtags per post, integrated naturally into the text or at the end
2. **LinkedIn:** 3-5 hashtags at the end of the post
3. **Reddit:** No hashtags (against culture). Use flair instead
4. **GitHub:** Use release tags, not hashtags
5. Never use `#follow4follow`, `#startup`, or generic spam hashtags

## Community Engagement Rules

### Response Guidelines

| Scenario                        | Response Time Target | Approach                                                    |
|---------------------------------|---------------------|-------------------------------------------------------------|
| Bug report                      | < 4 hours           | Acknowledge, ask for repro steps, assign to contributor     |
| Feature request                 | < 24 hours          | Thank them, add to discussion, link related issues          |
| Praise / testimonial            | < 2 hours           | Thank publicly, retweet/share, ask permission for quote     |
| Criticism                       | < 4 hours           | Acknowledge, don't be defensive, explain reasoning if needed |
| Question about setup            | < 8 hours           | Point to docs first, help directly if docs are insufficient |
| Competitor comparison           | Don't respond        | Let the product speak — never trash-talk competitors        |

### Engagement Voice

- **Be human.** Sign off with your name if it's a personal reply from a team member
- **Be grateful.** Every contributor, star, and issue is someone investing time in the project
- **Be transparent.** If something is broken, say it. If a feature is planned, share the timeline honestly
- **Never be defensive.** Criticism of open-source projects is feedback, not attacks
- **Celebrate others.** Retweet community projects built on TaskClaw. Share contributor content

### What NOT to Post

- Political opinions or controversial takes unrelated to tech
- Negative comments about competitors by name
- Unreleased features without the `[WIP]` or "coming soon" qualifier
- Promises with dates unless engineering has confirmed them
- Anything that could be perceived as AI hype without substance
- Content that uses the mascot inappropriately

## Content Calendar Framework

### Weekly Cadence (Example)

| Day         | Platform(s)      | Content Pillar                        | Format                    |
|-------------|------------------|---------------------------------------|---------------------------|
| **Monday**  | Twitter, LinkedIn | Product & Features                    | Feature spotlight + screenshot |
| **Tuesday** | Twitter           | Productivity & Pain Points            | Hot take / relatable post  |
| **Wednesday** | Twitter, Dev.to | Developer Education                   | Thread or short tutorial   |
| **Thursday** | Twitter, LinkedIn | Open Source & Community               | Contributor shoutout       |
| **Friday**  | Twitter           | Behind the Scenes                     | Dev update, WIP preview    |
| **Weekend** | Discord           | Community engagement                  | Q&A, casual conversation   |

### Monthly Milestones to Track

- GitHub stars milestone (every 100/500/1000)
- New integration launches
- Contributor count growth
- Release announcements (changelog posts)
- Community content reposts

### Release Day Playbook

1. **Pre-release (1-2 days before):** Teaser post — "Something new is coming to TaskClaw..."
2. **Release day:**
   - Main announcement thread on Twitter
   - GitHub release with detailed changelog
   - LinkedIn post with product screenshot
   - Discord announcement with @everyone
   - Dev.to article (if major feature)
3. **Post-release (1-3 days after):**
   - Tutorial thread on how to use the new feature
   - Repost community reactions
   - Address early feedback publicly

## Metrics & KPIs

### Awareness Metrics

| Metric                  | Target (Quarterly)  | Tool                     |
|-------------------------|---------------------|--------------------------|
| GitHub stars            | +500 per quarter    | GitHub                   |
| Twitter followers       | +300 per quarter    | Twitter Analytics         |
| Discord members         | +200 per quarter    | Discord                  |
| Impressions per post    | 2,000+ average      | Platform analytics        |

### Engagement Metrics

| Metric                  | Target              | Formula                                |
|-------------------------|---------------------|----------------------------------------|
| Engagement rate         | > 3%                | (likes + replies + retweets) / impressions |
| Click-through rate      | > 1.5%              | link clicks / impressions              |
| Reply rate              | > 0.5%              | replies / impressions                  |
| Community response time | < 4 hours avg       | Manual tracking                        |

### Conversion Metrics

| Metric                  | Target              | Tracking                               |
|-------------------------|---------------------|----------------------------------------|
| GitHub repo visits      | +20% per month      | GitHub Insights                        |
| Docs page views         | +15% per month      | Analytics                              |
| Discord join rate       | > 10% of link clicks | UTM tracking                          |
| Contributor conversion  | 5% of active Discord members | Manual tracking                |
