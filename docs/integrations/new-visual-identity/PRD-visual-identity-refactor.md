## Overview

### What

A comprehensive visual identity refactor of the TaskClaw frontend and communications to align with the new brand identity established in `docs/brand/`. This includes applying the new Clawster lobster mascot logo, updating the color palette to incorporate Claw Red (`#E63B3B`) alongside the existing purple-blue gradient, updating all metadata/SEO, and refactoring the landing page and dashboard UI to reflect the brand guidelines.

### Why

The current frontend carries residual "Onset" branding, placeholder metadata (`"Create Next App"`), no favicon, and a generic Lucide icon as the logo. The new TaskClaw visual identity — featuring the Clawster lobster mascot — gives the product a distinctive, memorable personality that differentiates it from every other task management tool. This refactor brings the codebase in line with the brand documentation.

### Who

- **Frontend/UI Team**: Implements all visual changes
- **Design Team**: Provides additional mascot poses and marketing assets
- **Content Team**: Updates copy to follow brand voice guidelines
- **DevOps**: Ensures new assets are optimized and served correctly

### 1.6 Update Brand Logo Component

**File:** `frontend/src/components/brand-logo.tsx`

The `BrandLogo` component needs a major update to support image-based logos (the TaskClaw mascot PNGs) with automatic dark/light switching.

**Changes required:**
- When `NEXT_PUBLIC_LOGO_TYPE=image`, render `<Image>` with the logo PNGs
- Add theme-aware logo switching (dark logo on dark bg, light logo on light bg)
- Support `variant="mascot-only"` for sidebar icon mode (just the lobster, no text)
- Keep the existing icon fallback for backwards compatibility

```tsx
// Pseudo-code for the updated component
function BrandLogo({ variant, className }) {
  const { theme } = useTheme();
  const logoSrc = theme === 'dark'
    ? '/images/logo/taskclaw_logo_dark.png'
    : '/images/logo/taskclaw_logo_light.png';

  if (variant === 'horizontal') {
    return <Image src={logoSrc} alt="TaskClaw" height={40} />;
  }

  if (variant === 'square' || variant === 'mascot-only') {
    return <Image src={logoSrc} alt="TaskClaw" height={32} width={32} objectFit="contain" />;
  }
}
```

## Technical Specifications

### Logo Integration Architecture

```
BrandLogo Component (brand-logo.tsx)
├── Image Mode (NEXT_PUBLIC_LOGO_TYPE=image) [DEFAULT NOW]
│   ├── Theme Detection (useTheme hook)
│   ├── Dark Theme → /images/logo/taskclaw_logo_dark.png
│   └── Light Theme → /images/logo/taskclaw_logo_light.png
├── Icon Mode (NEXT_PUBLIC_LOGO_TYPE=icon) [Backwards compat]
│   └── Lucide icon + text
└── Variants
    ├── horizontal: Full logo (mascot + "TASKCLAW" text)
    ├── square: Compact for sidebar collapsed state
    └── mascot-only: Lobster icon only (favicon, avatar)
```

### Image Optimization Pipeline

All logo images must be optimized for web delivery:

```
Source PNG →
  1. Crop to content bounds (remove excess whitespace)
  2. Generate sizes: 32, 64, 128, 192, 256, 512px
  3. Create WebP variants for browser support
  4. Create favicon.ico (multi-size: 16, 32, 48)
  5. Create apple-touch-icon (180x180)
  6. Create og-image (1200x630 with logo centered on brand background)
```

## Risks & Mitigations

| Risk | Impact | Likelihood | Mitigation |
|------|--------|------------|------------|
| Logo PNG quality at small sizes | Blurry favicons | Medium | Generate dedicated small-size renders, consider SVG mascot |
| Dark/light logo flash on theme switch | Visual glitch | Low | Use CSS `prefers-color-scheme` in addition to React state |
| Claw Red fails WCAG contrast on some backgrounds | Accessibility failure | Medium | Test all combinations; use `#C42D2D` (Shell Red) for text on light backgrounds |
| Breaking existing image-mode users | Config regression | Low | Keep backwards compatibility in BrandLogo component |
| SEO metadata changes cause temporary ranking disruption | Traffic dip | Low | Use 301 redirects for any URL changes; submit new sitemap immediately |
| Large PNG logos increase page load time | Performance | Medium | Compress PNGs with tinypng, serve WebP with fallback, lazy-load below fold |
